‚ö° Deep Dive: Building a Reactive Todo List with Angular Signals

Todo lists are a classic way to learn about state management. With Angular signals, you can create a todo list that updates instantly as you add items‚Äîno manual subscriptions, no complex state libraries, just pure reactivity.

A signal in Angular is a function that holds a value and notifies the UI whenever it changes. This makes it perfect for managing lists and collections in real time.

How This Example Works

  • Signal Declaration: The todo list is stored in a signal, e.g. todos = signal([]);
  • Adding Todos: When you enter a todo and click Add Todo, the signal is updated with the new item using todos.update(list => [...list, newTodo]).
  • Reactive UI: The template uses @for (item of todos(); track item) to display the list. Whenever the signal changes, Angular updates the UI automatically.
  • No Subscriptions Needed: Signals handle all reactivity for you. The UI stays in sync with the signal‚Äôs value in real time.
  • Instant Feedback: As you add todos, the list updates immediately, demonstrating fine-grained reactivity.

Why Use Signals for Lists?

  • Simplicity: No need for RxJS, Subjects, or manual subscriptions for simple list state.
  • Performance: Only the parts of the UI that use the signal are updated.
  • Readability: The code is easy to follow and maintain.
  • Fine-Grained Reactivity: Signals track exactly where they‚Äôre used, so updates are efficient and instant.
