üîÑ Deep Dive: Two-Way Binding with Signals and ngModel

Two-way binding is a core feature in Angular, allowing your UI and data model to stay in sync. With signals, you can make this even more reactive and efficient, especially for form inputs and user-driven state.

A signal in Angular is a function that holds a value and notifies the UI whenever it changes. When combined with ngModel, you get seamless two-way reactivity between your input fields and your application state.

How This Example Works

  • Signal Declaration: The username is stored in a signal, e.g. username = signal('');
  • Two-Way Binding: The input field uses [(ngModel)]="username" to bind the value both ways. When you type, the signal updates; when the signal changes, the input updates.
  • Reactive UI: The template uses {{ username() }} to display the value. Whenever the signal changes, Angular updates the UI automatically.
  • No Subscriptions Needed: Signals handle all reactivity for you. The UI and state stay in sync in real time.
  • Instant Feedback: As you type, the username display updates immediately, demonstrating fine-grained reactivity.

Why Use Signals with ngModel?

  • Simplicity: No need for RxJS, Subjects, or manual subscriptions for simple form state.
  • Performance: Only the parts of the UI that use the signal are updated.
  • Readability: The code is easy to follow and maintain.
  • Fine-Grained Reactivity: Signals track exactly where they‚Äôre used, so updates are efficient and instant.
