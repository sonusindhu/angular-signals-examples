Signal Example 16 - Readonly Signal (asReadonly)

This example demonstrates using asReadonly() to expose a readonly signal.
Writable signals can be updated and read, while readonly signals (created with asReadonly()) can only be read.
This is useful for sharing state with other components or templates without allowing them to modify it directly.
Try clicking the button to update the writable signal. The readonly signal will reflect the change, but cannot be updated directly.
